Getting started

Introduction

With FIR Controller you can configure and control a Red Pitaya board to act as a versatile frequency domain filter of analog signal from kHz to MHz range.

Filter is configured by loading a parametric FIR (finite impulse response) filter design onto the FPGA circuit that resides on the board. To achieve highest possible performance you can choose from different designs based on your needs. Without going into detail, the choice of parameters is a matter of trade-off between frequency range (called band) in which filter can operate and its performance, where, in simple terms, performance of the filter is given by the frequency delta between the frequencies where filter passes a signal (pass-band) and stops a signal (stop-band).

After a board is configured with filter design of choice, you can control operation of filter. This is done by specifying filter transmission function, which is a function of frequency with values corresponding to filter gain. This function has typically (and in our case) constant values over finite number of frequency ranges (bands). As previously mentioned, frequency range (band) where transmission function is equal to 1 is called pass-band and when is equal to 0 is called stop-band. However, transmission function can have any non-negative values. If its value (gain) is in the range from 0 to 1, signal is attenuated. If gain is bigger then 1, signal is amplified.

An actual transmission function of filter is only an approximation of specified transmission function. How good is this approximation depends on filter performance. You can inspect an actual transmission function on the plots.

To test different filter configurations and design transmission functions you don't have to connect to Red Pitaya. Rest of this section will quickly get you through basic steps to fully configure and specify filter and inspect the resulting transmission. More thorough explanation of particular steps is covered in next sections. You can also access appropriate section directly from application by clicking on question marks (?) near respective controls.

Workflow guide

The application window is divide into to parts. On the left side reside all controls responsible for setting up a filter, while on the right side you can inspect created filter. The left side is further divided into three groups, corresponding to the three steps necessary to create a working filter: configuration, specification and connection. While specification must be done after configuration, connection to Red Pitaya can be established in any moment. In this example we will follow the steps in the top-down order.

The most important setting of the configuration group is "Working band width". This setting determines the width of frequency range in which filter will operate and also its performance (the lower the better). Outside of this range transmission function (gain) will be always 0. Try setting this value to "500 kHz". You will notice, that in the specification group value of "Working band" changed to "0 - 500 kHz". You may wonder what is the difference between this settings? Forget for a moment about other controls in specification group and click on "working band" selection. You will see consequent ranges: "0 - 500 kHz", "500 - 1000 kHz", "1000 - 1500 kHz", and so on. Thanks to this option, if you are working with a high frequency signal, but with relatively narrow frequency spectrum, you can still use high performance filter configurations. To give you some numbers, if your signal frequency spectrum was in the range between 2.2 and 2.3 MHz, normally you would set "working band width" to 2.5 MHz. With possibility to choose higher band multiple you can set "Working band width" to 0.5 MHz which translates to 25 times increase in filter performance! The other control in configuration group can be left default. If you want to learn more about device configuration go here.

After choosing configuration, let's move on to specification group. Set "working band" to "0 - 500 kHz" for now. Leave the "window" control default and focus on "frequencies" and "gains" input fields. As mentioned before, transmission function is described by finite number of ranges with constant gain values. The simplest way to define such function is to specify (corner) frequencies at which it transitions from one gain to another and gain values for respective bands. Let's say we want a filter which blocks signal up to 200 kHz, passes it unchanged for frequencies from 200 to 210 kHz and decreases its amplitude by a factor of two from 210 to 500 kHz. In that case we need to set "frequencies" to "200 210" (kHz) and gains to "0 1 0.5". Note that you shouldn't enter band edge frequencies (0 and 500 kHz in this case), as there are set implicitly.

If you now hit "Calculate" button, you will see that orange line appeared on a plot on the right side that closely resembles specified transmission function.
